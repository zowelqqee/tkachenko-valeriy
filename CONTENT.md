# Как добавлять и редактировать события

Все события хранятся в одном файле: **`data/events.ts`**

---

## Добавить новое событие

Откройте `data/events.ts` и добавьте объект в конец массива `events`:

```typescript
{
  id: 'evt-009',                          // уникальный порядковый ID
  slug: 'название-события',              // уникальный slug — только латиница и дефисы
  title: 'НАЗВАНИЕ СОБЫТИЯ',             // заголовок на карточке и на странице
  shortTitle: 'Краткое название',        // необязательно, для узких блоков
  description: 'Описание события...',   // текст под заголовком в FeaturedEvent
  genre: 'Рок',                          // см. список жанров ниже
  date: '2026-10-15',                    // формат: YYYY-MM-DD
  time: '20:00',                         // формат: HH:MM
  datetimeISO: '2026-10-15T20:00:00+03:00', // тот же момент в ISO 8601 (нужен для сортировки)
  venue: 'Crocus City Hall',             // название площадки
  venueUrl: 'https://crocus-city-hall.ru', // ссылка на площадку (необязательно)
  city: 'Москва',                        // необязательно
  priceFrom: 3000,                       // минимальная цена, только число
  currency: 'RUB',                       // всегда RUB
  ageRating: '16+',                      // см. допустимые значения ниже
  buyTicketUrl: 'https://...',           // ссылка на покупку билета
  posterImage: '/images/posters/название-события.jpg', // необязательно, см. ниже
  heroImage: '/images/posters/название-события-hero.jpg', // необязательно (шире постера)
  galleryImages: [],                     // необязательно, [] если нет фото
  featured: false,                       // true — событие занимает слот «Ближайшее событие»
  published: true,                       // false — скрывает с сайта полностью
},
```

---

## Поля: допустимые значения

### `genre` — жанр

```
'Рок' | 'Джаз' | 'Поп' | 'Стендап' | 'Классика' | 'Блюз' | 'Электронная' | 'Фолк'
```

Жанр используется в фильтрах на странице Афиши и для цвета градиента на карточке.
Если жанра нет в списке — добавьте его в `types/event.ts` в тип `Genre`.

### `ageRating` — возрастной рейтинг

```
'0+' | '6+' | '12+' | '16+' | '18+'
```

### `featured` — главный слот

На главной странице есть один большой слот «Ближайшее событие».
Он автоматически занимается **ближайшим предстоящим** событием с `featured: true`.
Если несколько событий имеют `featured: true` — выбирается самое ближайшее.
Если ни у одного нет `featured: true` — берётся просто ближайшее предстоящее.

### `published` — видимость

- `true` — событие видно на сайте
- `false` — событие скрыто (черновик)

## Добавить постер

1. Подготовьте изображение в формате JPG или WebP, рекомендуемое соотношение сторон **3:4** (например, 600×800 px).
2. Поместите файл в папку:
   ```
   public/images/posters/
   ```
3. Назовите файл по slug события:
   ```
   public/images/posters/название-события.jpg
   ```
4. Укажите путь в поле `posterImage`:
   ```typescript
   posterImage: '/images/posters/название-события.jpg',
   ```

**Если постер не добавлен** — карточка покажет цветной градиент, соответствующий жанру события.

---

## Скрыть или снять событие

Чтобы временно скрыть событие — установите `published: false`.
Чтобы удалить полностью — просто удалите объект из массива.

---

## Изменить существующее событие

Найдите нужный объект по `id` или `title` и отредактируйте любое поле.
После сохранения файла сервер разработки обновится автоматически.

---

## Порядок событий

События автоматически сортируются по дате по полю `datetimeISO`.
Порядок объектов в массиве не важен.

---

## Перевести событие в архив

«Прошедшие» события не появляются в афише и на главной автоматически —
как только дата `datetimeISO` остаётся в прошлом, событие перестаёт показываться.
Удалять его не нужно: можно оставить в файле для истории.

---

## Структура файлов контента

| Файл | Что хранится |
|---|---|
| `data/events.ts` | Все события |
| `data/site.ts` | Тексты сайта, контакты, реквизиты |
| `data/legal.ts` | Страницы «Оплата» и «Возврат билетов» |
| `public/images/posters/` | Постеры событий |
| `public/images/gallery/` | Фотографии для раздела «Фото» |
